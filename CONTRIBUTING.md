## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/bilgee0517/cervyn-visualizer.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes (press F5 in VS Code)
6. Commit: `git commit -am 'Add some feature'`
7. Push: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Development Guidelines

### Code Style
- Use TypeScript
- Follow existing code formatting
- Add comments for complex logic
- Use meaningful variable names

### Testing
- Test your changes manually in VS Code
- Ensure the extension still works with different project types
- Check that layouts render correctly

### Commit Messages
Use clear and descriptive commit messages:
- `feat: add Python support`
- `fix: resolve layout issue with large graphs`
- `docs: update README with new features`

## Areas to Contribute

### High Priority
- Additional language support (Python, Java, Go)
- Search and filter functionality
- Performance improvements for large codebases

### Medium Priority
- Export to different formats (PNG, SVG, PlantUML)
- Custom node styling
- Keyboard shortcuts

### Future Features
- MCP server integration
- AI-powered insights
- Task tracker integration
